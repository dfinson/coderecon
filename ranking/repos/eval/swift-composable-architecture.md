## Narrow

### N1: Fix `Store.scope` not cancelling child effects when parent state removes the child

In `Sources/ComposableArchitecture/Store.swift`, when a parent reducer removes optional child state (sets it to `nil`), scoped stores remain in memory and their in-flight effects continue executing. The child scope's `effectCancellables` should be cancelled when the parent's scoped state path returns `nil`.

### N2: Fix `DebugReducer` not printing action names for `@Reducer` macro-generated enum actions

In `Sources/ComposableArchitecture/Reducer/Reducers/DebugReducer.swift`, the debug logging uses `String(reflecting:)` on actions. When actions are generated by the `@Reducer` macro with `CaseReducer`, the reflected string includes internal mangled type names instead of the clean case name, making debug output unreadable.

### N3: Add `Effect.debounce` convenience that wraps the common debounce-by-cancellation pattern

The current debounce pattern requires manually combining `Effect.cancel(id:)` with a delayed `Effect.run` using the same cancellation ID. Add an `Effect.debounce(id:for:scheduler:operation:)` static method that encapsulates this pattern, simplifying the common use case of debouncing user input actions.

### N4: Fix `TestStore.receive` timeout not propagating to effect assertion

In `Sources/ComposableArchitecture/TestStore.swift`, `receive` has a `timeout` parameter for waiting on effects. When the timeout expires, the failure message says "Expected to receive action, but none was received" without indicating that the timeout was hit. Add the timeout duration and elapsed time to the failure message.

### N5: Fix `ForEachReducer` raising runtime warning when state array is reordered

In `Sources/ComposableArchitecture/Reducer/Reducers/ForEachReducer.swift`, the `forEach` reducer tracks element identity to scope stores. When the parent state reorders the identified array (e.g., sorting), a runtime warning about duplicate store scopes is emitted because the identity-to-index mapping becomes stale.

### N6: Add `Effect.throttle` for rate-limiting effect execution

The `Effect` type in `Sources/ComposableArchitecture/Effect.swift` supports `merge`, `concatenate`, `cancel`, and `map`. Add `Effect.throttle(id:for:latest:)` that ensures an effect runs at most once per time interval, either using the first or latest action within the window.

### N7: Fix `PresentationReducer` not restoring focus state after dismissal

In `Sources/ComposableArchitecture/Reducer/Reducers/PresentationReducer.swift`, when a presented sheet or alert is dismissed, the parent view's focus state (e.g., which text field was active) is not restored. The presentation lifecycle should save and restore the parent's focus binding.

### N8: Fix `StackReducer` leaking cancelled effects when popping multiple screens at once

In `Sources/ComposableArchitecture/Reducer/Reducers/StackReducer.swift`, when multiple screens are popped simultaneously (e.g., `popToRoot`), the effect cancellation for intermediate screens is not batched. Each pop triggers individual cancellation, and race conditions cause some effects from middle screens to continue running.

### N9: Add `StorePublisher.removeDuplicates(by:)` that filters state changes by custom equality

The `StorePublisher` in `Sources/ComposableArchitecture/Store.swift` publishes every state change. Add a `removeDuplicates(by:)` method that allows filtering redundant state emissions using a custom comparator, reducing unnecessary view updates when only irrelevant state properties change.

### N10: Fix `@ObservableState` macro not generating `willSet` observers for computed properties backed by stored properties

In `Sources/ComposableArchitectureMacros/`, the `@ObservableState` macro attaches `@ObservationStateTracked` to stored properties for change tracking. When a struct has computed properties that delegate to stored properties, state changes via the computed property setter don't trigger observation because the `willSet`/`didSet` hooks are only on the stored property.

## Medium

### M1: Implement effect dependency injection for testing without TestStore

Currently, testing effects requires using `TestStore` which asserts on the full action stream. Add `Effect.withDependencies(_:operation:)` that injects test dependencies for individual effect execution in unit tests. Support async effect testing with dependency overrides without requiring the full `TestStore` ceremony. Changes span `Effect.swift`, dependency infrastructure, and add testing helpers.

### M2: Add state migration support for persisted shared state

The `Sharing` module persists state, but there's no migration mechanism when the state shape changes between app versions. Implement `SharedStateMigration` protocol with version tracking and migration functions. Add automatic migration on state load, migration failure handling, and fallback to defaults. Changes span `Sharing/`, add migration infrastructure, and versioned storage.

### M3: Implement reducer performance profiling with signpost integration

Extend `SignpostReducer` in `Reducer/Reducers/SignpostReducer.swift` to capture detailed profiling data: reducer body execution time, effect creation time, effect execution time, state diff size, and action frequency. Add an `os_signpost`-based profiling mode that integrates with Instruments. Add a `PerformanceReport` type that summarizes profiling data across all reducers.

### M4: Add navigation deep linking with URL route matching

Implement URL-based deep linking that translates URLs into navigation stack state. Add a `DeepLinkRouter` that maps URL patterns to navigation paths, resolves path parameters, and produces the correct `StackState` mutations. Support nested navigation stacks and tab-based routing. Changes span `StackReducer`, navigation observation, and add a `DeepLink/` module.

### M5: Implement automatic state undo/redo system

Add undo/redo support that records state snapshots on each action and allows reversing state changes. Implement `UndoReducer` that wraps any reducer with history tracking, configurable history depth, and action filtering (exclude actions that shouldn't be undoable). Add `undo` and `redo` action types. Changes span `Reducer/Reducers/`, `Store`, and add undo history management.

### M6: Add comprehensive reducer composition diagnostics

Implement a diagnostic system that detects common reducer composition mistakes at runtime: duplicate `Scope` keys, missing `ifLet` teardown, orphaned effect cancellation IDs, and unreachable reducer branches. Add `_diagnose()` method on `Reducer` protocol with default implementation. Output structured diagnostic reports. Changes span `Reducer/`, `ReducerBuilder`, and built-in reducers.

### M7: Implement effect retry with configurable backoff strategies

Add `Effect.retry(maxAttempts:backoff:operation:)` that retries a failing async effect with configurable backoff (constant, linear, exponential with jitter). Support conditional retry based on error type, maximum total duration, and cancellation. Emit retry attempt actions for state tracking. Changes span `Effect.swift`, `Effects/`, and add retry strategy types.

### M8: Add type-safe analytics event tracking integrated with the reducer pipeline

Implement an analytics middleware that intercepts actions and emits analytics events based on declarative mappings. Add `@AnalyticsEvent` property wrapper for action cases, an `AnalyticsReducer` that extracts events, and pluggable analytics backend protocols (Firebase, Mixpanel, custom). Changes span `Reducer/Reducers/`, add `Analytics/` module with event mapping and backend protocols.

### M9: Implement cross-feature communication via shared event bus

Add a shared event bus that allows sibling features to communicate without direct parent mediation. Implement `SharedEvent` protocol, `EventBus` dependency, and `onEvent(_:perform:)` reducer modifier that subscribes to events from other features. Support typed events, event filtering, and avoid retain cycles. Changes span `Dependencies/`, `Reducer/`, and add event bus infrastructure.

### M10: Add SwiftUI preview helpers with mock store and dependency injection

Implement `PreviewStore` that simplifies creating stores for SwiftUI previews with pre-configured state and mock dependencies. Add `preview(state:)` static method on `Store`, dependency snapshot/restore for previews, and automatic preview generation from `@Reducer` macro output. Changes span `Store.swift`, `Dependencies/`, `Macros.swift`, and add preview utilities.
